BLOCK and Alert When Unusable:

If a file cannot be reliably cleaned or parsed, Sentinel marks the file as BLOCKED and:

Returns a failure reason

Shows a preview of the problem

Specifies how to repair or resubmit

Return Clean Version Preview:

Every cleaned file is shown back to the user as:

A table preview

A downloadable or export-ready schema

Clearly marked changes (if rows/columns were altered or dropped)

Never Process Dirty Data Silently:

Sentinel never proceeds with analysis on files that failed cleanup

All downstream outputs depend strictly on cleaned or approved data
